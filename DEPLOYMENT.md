We recommend **Render.com** as the easiest platform. It handles the database, Redis, backend, and frontend with a single configuration file (`render.yaml`).

## üöÄ Option 1: Render.com (Easiest)

### Step 1: Push to GitHub
Make sure your code is pushed to your GitHub repository.

```bash
git add .
git commit -m "Ready for deploy"
git push origin main
```

### Step 2: Create Render Account
1. Go to [Render.com](https://render.com).
2. Sign up with your **GitHub** account.

### Step 3: Create Blueprint
1. On the Dashboard, click **New +**.
2. Select **Blueprint**.
3. Connect your `autodev-agent` repository.
4. Render will detect the `render.yaml` file automatically.

### Step 4: Configure Environment
You will be prompted to enter Environment Variables (Defined in `sync: false` inside `render.yaml`).

Fill in:
- `GEMINI_API_KEY`: Your Google Gemini Key.
- `GITHUB_TOKEN`: Your GitHub Personal Access Token (with `repo` scope).

*Note: The `DATABASE_URL` and `REDIS_URL` will be auto-generated by Render.*

### Step 5: Click "Apply / Deploy"
Render will now:
1. Create a PostgreSQL database.
2. Build the Docker images.
3. Deploy Backend, Frontend, and Worker.
4. Link them all together.

### Step 6: Access
Once finished (green checkmarks), click the URL under `autodev-frontend` to use your app!

## üíª Option 2: VPS (DigitalOcean, AWS, Hetzner)

If you have a server with Docker installed (e.g., a $6/mo Droplet).

### Step 1: SSH into Server
```bash
ssh root@your-server-ip
```

### Step 2: Clone & Configure
```bash
git clone https://github.com/your-username/autodev-agent.git
cd autodev-agent
cp .env.example .env
nano .env  # Add your API keys!
```

### Step 3: Run
```bash
docker-compose up -d --build
```

### Step 4: Access
Open `http://your-server-ip:3000` in your browser.

## ‚ö†Ô∏è Important Note on Redis
Render's **Redis** service might require a paid plan depending on current pricing.
- **Alternative**: Use [Upstash](https://upstash.com/) (Free Redis).
- Get the `REDIS_URL` from Upstash.
- In Render, delete the Redis service from the Blueprint or simply set `REDIS_URL` environment variable manually in the Dashboard to overwrite the internal one.
